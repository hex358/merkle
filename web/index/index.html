How CertumTree Works

CertumTree is a zero-trust data verification layer designed to prevent tampering and fraud. When a digital object - such as a message, file, or transaction - is registered through the API, it is reduced to a cryptographic hash. The system never stores the raw content, only these hashes, which act as irreversible digital fingerprints.


All hashes are organized into an append-only Merkle Mountain Range, ensuring that once a record is added, it cannot be modified or deleted. Each new insertion extends the chain of proofs while preserving the full history of prior states.


To guarantee transparency, root hashes - the condensed representation of the entire dataset - are publicly available. This allows independent auditors or third parties to confirm that no hidden modifications occurred between checkpoints.


Under the hood, CertumTree is powered by a custom high-performance key-value engine, derived from LMDB - HexDB. HexDB is optimized for millions of inserts per second, enabling CertumTree to handle large-scale verification workloads without sacrificing speed or consistency.


The result is a system where authenticity can be proven both ways: services can issue cryptographic proofs of integrity, and clients can independently validate them without relying on trust in a central authority.


The CertumTree API makes integration straightforward. Applications can register new records, request inclusion proofs, and verify data integrity with just a few endpoints. Designed to be lightweight and stateless, the API can be embedded into existing systems with minimal effort, enabling real-time verification for messages, documents, financial transactions, or any other digital objects.


The project is fully open source under the permissive MIT license, encouraging community contributions, transparency, and independent adoption. Many new features and improvements (such as WebRTC support for even faster verifications) will continue to be added as the ecosystem grows.
